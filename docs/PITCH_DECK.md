
## 第 6 页：核心算法——多模态交通套利搜索

**问题建模**：多模态交通网络
- 节点 = 城市/枢纽/口岸站点
- 边 = 一段交通（飞机/高铁/卧铺/地铁/过关），带多维成本（价格、耗时、换乘、舒适度）

**算法**：多目标 Dijkstra 变种
- 综合成本 = w₁·价格 + w₂·时间 + w₃·换乘次数 + w₄·舒适度
- 用户偏好（省钱/时间/均衡）调整权重
- 输出 Top-K 条 Pareto 候选路径，再与直达基准对比算省钱额

**关键差异**：不是最短路，是"省钱空间最大"的路。
